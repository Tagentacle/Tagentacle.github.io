🌟 Core Philosophy

Everything is a Pkg — Thorough modularization of system capabilities, inherited from ROS 2's most fundamental software organization philosophy.

🤖

Agent Package

Each agent as an independent package, containing behavior logic, prompt templates, state management, and communication interfaces.

🔧

Tool / Service Package

Encapsulates tools or services that agents call — database access, web scraping — with MCP protocol for plug-and-play integration.

📐

Interface Package

Defines cross-node communication contracts via JSON Schema, ensuring packages from different developers "speak the same language".

🚀

Bringup Package

System startup and configuration: workspace package sets, node launch parameters, environment credentials, and topology orchestration.

♻️

High Reusability

Mature tool/agent packages migrate seamlessly across projects like LEGO bricks.

🔒

Dependency Isolation

Per-package independent Python venvs, eliminating dependency conflicts entirely.

📦

Black-Box Development

Focus on input/output contracts — no need to know the internal model or framework.

🏗️ Robustness

Distributed Bus vs Monolithic Gateway
The Problem

Running 50 skills in one Node.js process — a memory leak in one reboots everything.

Tagentacle's Solution

Every Node is a separate process. A crashed "Twitter Scraper" never affects your "SRE Agent". Rust-based Broker provides high-concurrency routing that never sleeps.

⚡ Autonomy

Event-Driven Lifeforms vs Task-Based Tools
The Problem

CLI tools only act when commanded. They are "dead" when the command ends.

Tagentacle's Solution

Agents are "alive" 24/7. They watch logs at 3 AM, detect crashes, and fix issues autonomously — a Digital Persona, not a mere tool.

🎛️ Professional Grade

Mission Control vs Chat Bubbles
The Problem

Most frameworks squeeze everything into Telegram or WhatsApp bubbles.

Tagentacle's Solution

Exposes a raw data bus for "Mission Control" dashboards — real-time topology, CPU graphs, and interactive editors driven by the same message bus.

🔌 Python SDK

Dual-layer design — Simple API for quick integration, Node API for full lifecycle management.

from tagentacle_py import Node
import asyncio

async def main():
    node = Node("sensor_node")
    await node.connect()

    @node.subscribe("/data/temperature")
    async def on_temp(msg):
        print(f"Temperature: {msg['payload']['value']}°C")

    await node.publish("/status/online", {"node": "sensor_node"})
    await node.spin()

asyncio.run(main())
from tagentacle_py import LifecycleNode

class AliceAgent(LifecycleNode):
    def on_configure(self, config):
        self.api_key = config.get("api_key")
        self.allowed_tools = config.get("tools", [])

    def on_activate(self):
        self.subscribe("/task/inbox", self.handle_task)

    async def handle_task(self, msg):
        result = await self.call_service("/tool/search", msg["payload"])
        await self.publish("/task/result", result)

    def on_shutdown(self):
        self.logger.info("Alice shutting down gracefully.")
# The MCP-Publish bridge exposes bus topics as MCP tools
# Built-in MCP Server Node abstracts publish() as an MCP Tool

from mcp import ClientSession
from tagentacle_py.mcp import TagentacleClientTransport

# Connect through Tagentacle bus instead of stdio
transport = TagentacleClientTransport(node, server_node_id="sqlite_server")
async with ClientSession(transport) as session:
    await session.initialize()
    result = await session.call_tool("query", {"sql": "SELECT * FROM users"})

🛠️ MCP Integration

Bus-as-Transport — solving MCP Session cross-process serialization with elegant architecture.

🏠 Session Localization

MCP Client/Server Sessions stay in node memory — never transmitted cross-process. Sessions are local, performance is real.

🚌 Bus Traffic Forwarding

Custom TagentacleTransport wraps JSON-RPC into Service requests, achieving transparent and efficient tool calls in distributed environments.

🔀 Dual-Track Integration

MCP payloads are simultaneously mirrored to a dedicated Topic (/mcp/traffic) for non-intrusive observation and debugging.

↔️ Bidirectional Calling

Full MCP spec support including Sampling (Server calling Agent). Agent nodes register /rpc services for callbacks.

👁️ Full Observability

All tunneled traffic auto-mirrored to /mcp/traffic Topic. Any node can audit tool-calling flows without intrusive proxies.

🔌 Seamless SDK

Connect through the bus just like stdio. Swap transport layer and everything works — no code changes needed in your MCP logic.

📜 Communication Protocol

TCP 19999, newline-delimited JSON (JSON Lines). Simple, fast, universal.

📡 Topics

Subscribe
{"op": "subscribe", "topic": "/chat/global", "node_id": "alice_node"}
Publish
{"op": "publish", "topic": "/chat/global", "sender": "bob_node", "payload": {"text": "Hello!"}}
Message (Push)
{"op": "message", "topic": "/chat/global", "sender": "bob_node", "payload": {"text": "Hello!"}}

🔁 Services

Advertise
{"op": "advertise_service", "service": "/tool/read_file", "node_id": "fs_node"}
Call
{"op": "call_service", "service": "/tool/read_file", "request_id": "req-1", "payload": {"path": "a.txt"}}
Response
{"op": "service_response", "service": "/tool/read_file", "request_id": "req-1", "payload": {"content": "..."}}

🛠️ CLI Toolchain

The primary developer interface — manage daemons, packages, environments, and services from the command line.

tagentacle daemon
Start the local TCP message bus
tagentacle run --pkg <dir>
Activate package .venv and launch its Node
tagentacle launch <config.toml>
Orchestrate multi-node from topology config
tagentacle topic echo <topic>
Subscribe and print real-time messages
tagentacle service call <srv> <json>
Test a service from the command line
tagentacle bridge --mcp <cmd>
Bridge external MCP Server (stdio) to bus
tagentacle setup dep --pkg <dir>
Run uv sync for a single package
tagentacle setup dep --all <ws>
Install all packages, generate install/ structure
tagentacle doctor
Health check — daemon status, node connectivity
# Initialize entire workspace
tagentacle setup dep --all .
# → runs uv sync in each package
# → creates install/src/<pkg>/.venv symlinks
# → generates install/setup_env.bash

# Source environment (adds all .venvs to PATH)
source install/setup_env.bash

# Clean up
tagentacle setup clean --workspace .

🚀 Quick Start

Get up and running in three steps.

1

Start the Daemon (Rust Core)

cd tagentacle

cargo run -- daemon

2

Set Up the Workspace (uv)

cd tagentacle-py

tagentacle setup dep --all ..

3

Run a Node

tagentacle run --pkg examples/src/mcp_server_pkg
