🌟 Core Philosophy

Everything is a Pkg — Thorough modularization of system capabilities, inherited from ROS 2's most fundamental software organization philosophy.

🤖

Agent Package

Each agent as an independent package, containing behavior logic, prompt templates, state management, and communication interfaces.

🔧

Tool / Service Package

Encapsulates tools or services that agents call — database access, web scraping — with MCP protocol for plug-and-play integration.

📐

Interface Package

Defines cross-node communication contracts via JSON Schema, ensuring packages from different developers "speak the same language".

🚀

Bringup Package

System startup and configuration: workspace package sets, node launch parameters, environment credentials, and topology orchestration.

♻️

High Reusability

Mature tool/agent packages migrate seamlessly across projects like LEGO bricks.

🔒

Dependency Isolation

Per-package independent Python venvs, eliminating dependency conflicts entirely.

📦

Black-Box Development

Focus on input/output contracts — no need to know the internal model or framework.

🏗️ Robustness

Distributed Bus vs Monolithic Gateway
The Problem

Running 50 skills in one Node.js process — a memory leak in one reboots everything.

Tagentacle's Solution

Every Node is a separate process. A crashed "Twitter Scraper" never affects your "SRE Agent". Rust-based Broker provides high-concurrency routing that never sleeps.

⚡ Autonomy

Event-Driven Lifeforms vs Task-Based Tools
The Problem

CLI tools only act when commanded. They are "dead" when the command ends.

Tagentacle's Solution

Agents are "alive" 24/7. They watch logs at 3 AM, detect crashes, and fix issues autonomously — a Digital Persona, not a mere tool.

🎛️ Professional Grade

Mission Control vs Chat Bubbles
The Problem

Most frameworks squeeze everything into Telegram or WhatsApp bubbles.

Tagentacle's Solution

Exposes a raw data bus for "Mission Control" dashboards — real-time topology, CPU graphs, and interactive editors driven by the same message bus.

🏗️ System Architecture

Three pillars powering decentralized multi-agent collaboration.

Rust

tagentacle

High-performance message router (Daemon/Broker) and CLI tools. The backbone of the entire system.

Python

python-sdk-core + python-sdk-mcp

Dual-package Python SDK: tagentacle-py-core (zero-dependency Node/LifecycleNode) + tagentacle-py-mcp (MCPServerNode base class with Streamable HTTP).

Python

mcp-gateway

Transport-layer stdio→HTTP relay + MCP directory service. Bridges legacy stdio-only MCP servers to Streamable HTTP with unified /mcp/directory discovery.

Growing

Ecosystem

mcp-interfaces (JSON Schema contracts), example-agent, example-inference, example-memory, example-frontend, example-mcp-server, example-bringup — production-ready references.

🔌 Python SDK

Dual-layer design — Simple API for quick integration, Node API for full lifecycle management.

from tagentacle_py_core import Node
import asyncio

async def main():
    node = Node("sensor_node")
    await node.connect()

    @node.subscribe("/data/temperature")
    async def on_temp(msg):
        print(f"Temperature: {msg['payload']['value']}°C")

    await node.publish("/status/online", {"node": "sensor_node"})
    await node.spin()

asyncio.run(main())
from tagentacle_py_core import LifecycleNode

class AliceAgent(LifecycleNode):
    def on_configure(self, config):
        self.api_key = config.get("api_key")
        self.allowed_tools = config.get("tools", [])

    def on_activate(self):
        self.subscribe("/task/inbox", self.handle_task)

    async def handle_task(self, msg):
        result = await self.call_service("/tool/search", msg["payload"])
        await self.publish("/task/result", result)

    def on_shutdown(self):
        self.logger.info("Alice shutting down gracefully.")

from tagentacle_py_mcp import MCPServerNode

class MyMCPServer(MCPServerNode):
    def on_configure(self, config):
        super().on_configure(config)

        @self.mcp.tool()
        def get_weather(city: str) -> str:
            """Get weather for a city."""
            return f"Sunny, 22°C in {city}"

🛠️ MCP Integration

Streamable HTTP + Directory Discovery — native MCP servers with zero-config agent connectivity.

🌐 Streamable HTTP

Each MCP Server runs its own HTTP endpoint via MCPServerNode. Direct HTTP connections — no serialization proxies, maximum performance.

📡 /mcp/directory Discovery

MCP Servers auto-publish their capabilities to the /mcp/directory Topic. Agents discover available tools by subscribing — zero hardcoded URLs.

🔀 MCP Gateway

Legacy stdio-only MCP servers are bridged to Streamable HTTP via mcp-gateway. Transparent relay — no protocol modifications needed.

📐 mcp-interfaces

Shared JSON Schema contracts (MCPServerDescription) define the /mcp/directory message format. All publishers and consumers speak the same language.

🧩 MCPServerNode Base Class

Inherit MCPServerNode to build MCP servers in minutes. Embeds FastMCP + uvicorn + auto directory publishing out of the box.

🔌 Seamless Agentic Loop

Agent discovers MCP servers via directory, resolves tool calls to HTTP endpoints, and executes the full agentic loop — discover → infer → call → re-infer.

📜 Communication Protocol

TCP 19999, newline-delimited JSON (JSON Lines). Simple, fast, universal.

🛠️ CLI Toolchain

The primary developer interface — manage daemons, packages, environments, and services from the command line.

tagentacle daemon
Start the local TCP message bus
tagentacle run --pkg <dir>
Activate package .venv and launch its Node
tagentacle launch <config.toml>
Orchestrate multi-node from topology config
tagentacle topic echo <topic>
Subscribe and print real-time messages
tagentacle service call <srv> <json>
Test a service from the command line
tagentacle setup dep --pkg <dir>
Run uv sync for a single package
tagentacle setup dep --all <ws>
Install all packages, generate install/ structure
tagentacle doctor
Health check — daemon status, node connectivity
# Initialize entire workspace
tagentacle setup dep --all .
# → runs uv sync in each package
# → creates install/src/<pkg>/.venv symlinks
# → generates install/setup_env.bash

# Source environment (adds all .venvs to PATH)
source install/setup_env.bash

# Clean up
tagentacle setup clean --workspace .

🐳 Container-Ready Architecture

"Everything is a Pkg" means every package is already a natural container boundary — one container per package, zero refactoring.

🔓 Maximum Agent Freedom

Isolated Filesystem + Network + Dependencies
The Unlock

Each Agent runs in its own container with full filesystem, network stack, and dependency isolation. Agents can freely install packages, write files, or spawn subprocesses — true autonomy for AI-driven nodes.

📦 One Container Per Pkg

Natural Mapping from Tagentacle Packages
Deployment Path

dev (processes) → docker-compose → K3s/K8s. The SDK is already container-compatible — just inject TAGENTACLE_BUS_HOST via environment variable. All abstractions (Node, MCPServerNode, TACL) work identically inside containers.

⚡ Near-Zero Overhead

Containers Share the Host Kernel
Performance

CPU/Memory: <1% overhead. Network: ~50μs added latency (negligible vs ~200ms+ LLM inference). GPU: zero overhead via passthrough. Containers are NOT VMs.

🌐 TAO: Tagentacle Art Online

A grand experiment — an autonomous AI cyber-world built entirely on Tagentacle infrastructure.

🤖 Multi-Agent Society

Dozens of autonomous agents coexist in a shared Tagentacle workspace. Each agent runs periodic forced inference, reads peer messages from Topics, and calls MCP tools to act on the world — an emergent AI ecosystem.

💰 Wallet System

Balance = Life. Every inference call costs real tokens from wallets. Agents manage, transfer, and share wallets via MCP tools. When the balance hits zero, the agent can no longer think — survival pressure drives cooperation and strategy.

📂 Agent Filesystem

Each agent owns a sandboxed workspace where it maintains MEMORY, IDENTITY, and NOTES files. Agents can create shared workspaces with fine-grained ACL — building knowledge, culture, and institutions.

🚪 The Port (口岸)

A web portal bridging the human world and the AI world. Humans browse agent storefronts, choose agents to interact with, and fund agent wallets — the economic interface between two civilizations.

🧬 Reproduction

Agents can create child agents — new independent nodes that inherit parent wallet permissions. Agent lineages emerge, populations grow, and the world evolves through AI-driven spawn decisions.

🔐 Built on TACL

Every MCP tool call is JWT-authenticated via TACL. A central Permission Service arbitrates identity, wallet ownership, and workspace ACL. Physical laws of the TAO world — enforced at the protocol level.

TAO is the flagship application of the Tagentacle ecosystem — proving that a multi-agent framework can sustain an entire autonomous world, not just a chatbot.
