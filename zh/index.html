🌟 核心理念

一切皆 Pkg —— 将系统能力彻底模块化，这继承自 ROS 2 最根本的软件组织哲学。

🤖

Agent 包

每个 Agent 作为独立包，包含行为逻辑、Prompt 模板、状态管理和通信接口。

🔧

工具 / 服务包

封装 Agent 调用的工具或服务 —— 数据库访问、网页抓取 —— 通过 MCP 协议实现即插即用。

📐

接口包

通过 JSON Schema 定义跨节点通信契约，确保不同开发者的包"说同一种语言"。

🚀

Bringup 包

系统启动与配置：工作区包集合、节点启动参数、环境凭证和拓扑编排。

♻️

高复用性

🔒

依赖隔离

每个包独立的 Python venv，彻底消除依赖冲突。

📦

黑盒开发

只关注输入/输出契约 —— 无需了解内部使用的模型或框架。

🏗️ 健壮性

分布式总线 vs 单体网关
痛点

50 个技能挤在一个 Node.js 进程里 —— 一个内存泄漏，全体重启。

Tagentacle 方案

每个 Node 是独立进程。"推特爬虫"崩溃绝不影响你的"SRE Agent"。Rust 实现的 Broker 提供高并发路由，永不宕机。

⚡ 自主性

事件驱动生命体 vs 任务型工具
痛点

CLI 工具只在被命令时才行动。命令结束它就"死"了。

Tagentacle 方案

Agent 全天候"活着"。凌晨 3 点监控日志、检测崩溃并自主修复 —— 一个数字人格，而非单纯的工具。

🎛️ 专业级

指挥中心 vs 聊天气泡
痛点

大多数框架把一切塞进 Telegram 或 WhatsApp 的气泡里。

Tagentacle 方案

暴露原始数据总线，支持"指挥中心"仪表盘 —— 实时拓扑、CPU 图表和交互式编辑器，都由同一条消息总线驱动。

🔌 Python SDK

双层设计 —— Simple API 快速集成，Node API 完整生命周期管理。

from tagentacle_py_core import Node
import asyncio

async def main():
    node = Node("sensor_node")
    await node.connect()

    @node.subscribe("/data/temperature")
    async def on_temp(msg):
        print(f"温度: {msg['payload']['value']}°C")

    await node.publish("/status/online", {"node": "sensor_node"})
    await node.spin()

asyncio.run(main())
from tagentacle_py_core import LifecycleNode

class AliceAgent(LifecycleNode):
    def on_configure(self, config):
        self.api_key = config.get("api_key")
        self.allowed_tools = config.get("tools", [])

    def on_activate(self):
        self.subscribe("/task/inbox", self.handle_task)

    async def handle_task(self, msg):
        result = await self.call_service("/tool/search", msg["payload"])
        await self.publish("/task/result", result)

    def on_shutdown(self):
        self.logger.info("Alice 正在优雅关闭。")
# 组合 MCPServerComponent 创建基于 Streamable HTTP 的 MCP 服务器
# 自动发布到 /mcp/directory 供 Agent 发现

from tagentacle_py_core import LifecycleNode
from tagentacle_py_mcp import MCPServerComponent

class MyMCPServer(LifecycleNode):
    def on_configure(self, config):
        self.mcp_server = MCPServerComponent(self, config)

        @self.mcp_server.mcp.tool()
        def get_weather(city: str) -> str:
            """Get weather for a city."""
            return f"Sunny, 22°C in {city}"

🛠️ MCP 集成

Streamable HTTP + 目录发现 —— 原生 MCP 服务器，零配置 Agent 连接。

🌐 Streamable HTTP

每个 MCP Server 通过 MCPServerComponent 运行自己的 HTTP 端点。直连 HTTP —— 无序列化代理，性能极致。

📡 /mcp/directory 发现

MCP Server 自动将能力发布到 /mcp/directory Topic。Agent 通过订阅发现可用工具 —— 零硬编码 URL。

🔀 MCP Gateway

传统 stdio MCP 服务器通过 mcp-gateway 桥接为 Streamable HTTP。透明中继 —— 无需修改协议。

📐 mcp-interfaces

共享 JSON Schema 契约（MCPServerDescription）定义 /mcp/directory 消息格式。所有发布者和消费者说同一种语言。

🧩 MCPServerComponent

将 MCPServerComponent 组合到任意 LifecycleNode 即可分钟内构建 MCP 服务器。内置 FastMCP + uvicorn + 自动目录发布，开箱即用。

🔌 无缝 Agentic Loop

Agent 通过目录发现 MCP 服务器，将工具调用解析为 HTTP 端点，执行完整的 agentic loop —— 发现 → 推理 → 调用 → 再推理。

🔐 TACL：访问控制层

MCP 级别的 JWT 认证与授权 — 完全在 SDK 中实现，Daemon 不参与。

🏛️ 三角色

签发者（TACLAuthority）— SQLite 支撑的 Agent 注册中心，签发带工具级授权的 JWT。
验证者（MCPServerComponent + auth_required=True）— 每次请求验证 Bearer JWT。
携带者（AuthMCPClient）— 认证后获取 JWT，附加到所有 MCP 请求。

🎫 JWT 负载

agent_id — Agent 唯一标识。
tool_grants — 按服务器的工具白名单（{server_id: [tool_names]}）。
space — 执行环境绑定（如 Docker 容器）。
iat / exp — 签发时间 / 过期时间（默认 24h）。

📦 space 声明

将 Agent 身份绑定到隔离执行环境。管理员注册 register_agent(space="container_1")，JWT 携带此绑定，MCP 服务器（如 shell-server）将命令路由到绑定的容器 — 密码学证明，而非自我声明。

🔑 零外部依赖

JWT 签名/验证使用纯 Python 标准库（HS256 via hmac + hashlib）。共享密钥通过 TAGENTACLE_AUTH_SECRET 环境变量。CallerIdentity 通过 Python contextvars 按请求设置。

🎯 工具级粒度

授权粒度到每个服务器的工具级别 — 而非“全有或全无”。Agent A 可以调用 shell-server 的 exec_command，但不能调用 wallet-server 的 transfer。在 MCP 协议层实现细粒度控制。

⚙️ 可选启用

认证默认关闭。MCPServerComponent(auth_required=False)（默认值）接受所有调用者。按需启用安全控制 — 不需要时零开销。Daemon 始终不参与认证。

📜 通信协议

TCP 19999，换行分隔 JSON（JSON Lines）。简洁、快速、通用。

📡 Topic

订阅
{"op": "subscribe", "topic": "/chat/global", "node_id": "alice_node"}
发布
{"op": "publish", "topic": "/chat/global", "sender": "bob_node", "payload": {"text": "你好！"}}
消息（推送）
{"op": "message", "topic": "/chat/global", "sender": "bob_node", "payload": {"text": "你好！"}}

🔁 Service

注册
{"op": "advertise_service", "service": "/tool/read_file", "node_id": "fs_node"}
调用
{"op": "call_service", "service": "/tool/read_file", "request_id": "req-1", "payload": {"path": "a.txt"}}
响应
{"op": "service_response", "service": "/tool/read_file", "request_id": "req-1", "payload": {"content": "..."}}

🐳 天然容器化架构

“一切皆 Pkg” 意味着每个包天然就是容器的边界 —— 一容器一包，零重构。

🔓 Agent 的最大自由度

独立文件系统 + 网络栈 + 依赖树
解锁能力

每个 Agent 运行在自己的容器中，拥有完全隔离的文件系统、网络栈和依赖环境。Agent 可以自由安装库、写文件、启动子进程 —— 真正的 AI 节点自治。

📦 一容器一包

从 Tagentacle Pkg 到容器的自然映射
部署路径

开发（进程） → docker-compose → K3s/K8s。SDK 天然兼容容器化 —— 只需通过环境变量注入 TAGENTACLE_BUS_HOST。所有抽象（Node、MCPServerComponent、TACL）在容器内行为一致。

⚡ 近零开销

容器共享宿主机内核
性能

CPU/内存：<1% 开销。网络：增加约 50μs 延迟（相比 ~200ms+ 的 LLM 推理延迟可忽略）。GPU：通过直通实现零开销。容器不是虚拟机。

🔧 容器生态包

🏗️ 容器编排器

LifecycleNode 通过总线管理 Docker 容器 — 不是 Daemon 核心的一部分。提供 6 个总线服务：/containers/create、stop、remove、list、inspect、exec。自动标记 tagentacle.managed=true，自动注入 TAGENTACLE_DAEMON_URL。

🖥️ Shell Server

LifecycleNode + MCPServerComponent 暴露 exec_command，支持 TACL space 感知的动态路由。解析优先级：JWT space → 静态容器 → 本地 subprocess。单实例同时服务多个 Agent，每个请求根据 JWT 声明路由到各自容器。

🔗 端到端流程

1. 管理员注册 Agent 并绑定 space → 2. 编排器创建容器 → 3. Agent 认证（JWT 携带 space）→ 4. Shell-server 将 exec_command 路由到绑定容器。容器绑定以 JWT 密码学证明。

🌐 TAO: Tagentacle Art Online

一场宏大实验 —— 完全基于 Tagentacle 基础设施构建的自主 AI 赛博世界。

🤖 多 Agent 社会

数十个自主 Agent 共存于同一个 Tagentacle 工作区。每个 Agent 定期触发强制推理，从 Topic 读取同伴消息，通过 MCP 工具作用于世界 —— 一个涌现式 AI 生态系统。

💰 钱包系统

余额 = 生命。每次推理调用都消耗钱包中的真实 Token。Agent 通过 MCP 工具管理、转账、共享钱包。余额归零，Agent 就无法思考 —— 生存压力驱动合作与策略。

📂 Agent 文件系统

每个 Agent 拥有沙箱化的工作区，维护 MEMORY、IDENTITY、NOTES 等文件。Agent 可以创建共享工作区并设置细粒度权限 —— 构建知识、文化与制度。

🚪 口岸

一个连接人类世界和 AI 世界的网页门户。人类用户浏览 Agent 店铺、选择 Agent 交互、为 Agent 钱包充值 —— 两个文明之间的经济接口。

🧬 繁殖

Agent 可以创建子 Agent —— 新的独立节点，继承父代钱包权限。Agent 血统形成，种群增长，世界通过 AI 驱动的繁殖决策实现演化。

🔐 基于 TACL

每个 MCP 工具调用都经 TACL JWT 认证。中心权限服务仲裁身份、钱包所有权和工作区 ACL。TAO 世界的物理定律 —— 在协议层强制执行。

TAO 是 Tagentacle 生态的旗舰应用 —— 证明一个多 Agent 框架可以支撑整个自主世界，而不仅仅是一个聊天机器人。
