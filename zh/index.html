Tagentacle
The ROS of AI Agents

一个去中心化、配置集中化、生产级的多 Agent 框架。 融合 ROS 2 设计范式、MCP 协议与 Rust 性能。

🌟 核心理念

一切皆 Pkg —— 将系统能力彻底模块化，这继承自 ROS 2 最根本的软件组织哲学。

🤖

Agent 包

每个 Agent 作为独立包，包含行为逻辑、Prompt 模板、状态管理和通信接口。

🔧

工具 / 服务包

封装 Agent 调用的工具或服务 —— 数据库访问、网页抓取 —— 通过 MCP 协议实现即插即用。

📐

接口包

通过 JSON Schema 定义跨节点通信契约，确保不同开发者的包"说同一种语言"。

🚀

Bringup 包

系统启动与配置：工作区包集合、节点启动参数、环境凭证和拓扑编排。

♻️

高复用性

成熟的工具/Agent 包可以像乐高积木一样在项目间无缝迁移。

🔒

依赖隔离

每个包独立的 Python venv，彻底消除依赖冲突。

📦

黑盒开发

只关注输入/输出契约 —— 无需了解内部使用的模型或框架。

⚔️ 为什么选择 Tagentacle？

为下一代 AI 打造的工业级基础设施，超越单体网关和命令行工具。

架构Node.js 单体单进程 CLI分布式微服务 (Rust)

🏗️ 健壮性

分布式总线 vs 单体网关
痛点

50 个技能挤在一个 Node.js 进程里 —— 一个内存泄漏，全体重启。

Tagentacle 方案

每个 Node 是独立进程。"推特爬虫"崩溃绝不影响你的"SRE Agent"。Rust 实现的 Broker 提供高并发路由，永不宕机。

⚡ 自主性

事件驱动生命体 vs 任务型工具
痛点

CLI 工具只在被命令时才行动。命令结束它就"死"了。

Tagentacle 方案

Agent 全天候"活着"。凌晨 3 点监控日志、检测崩溃并自主修复 —— 一个数字人格，而非单纯的工具。

🎛️ 专业级

指挥中心 vs 聊天气泡
痛点

大多数框架把一切塞进 Telegram 或 WhatsApp 的气泡里。

Tagentacle 方案

暴露原始数据总线，支持"指挥中心"仪表盘 —— 实时拓扑、CPU 图表和交互式编辑器，都由同一条消息总线驱动。

🏗️ 系统架构

三大支柱，驱动去中心化多 Agent 协作。

🔌 Python SDK

双层设计 —— Simple API 快速集成，Node API 完整生命周期管理。

from tagentacle_py import Node
import asyncio

async def main():
    node = Node("sensor_node")
    await node.connect()

    @node.subscribe("/data/temperature")
    async def on_temp(msg):
        print(f"温度: {msg['payload']['value']}°C")

    await node.publish("/status/online", {"node": "sensor_node"})
    await node.spin()

asyncio.run(main())
from tagentacle_py import LifecycleNode

class AliceAgent(LifecycleNode):
    def on_configure(self, config):
        self.api_key = config.get("api_key")
        self.allowed_tools = config.get("tools", [])

    def on_activate(self):
        self.subscribe("/task/inbox", self.handle_task)

    async def handle_task(self, msg):
        result = await self.call_service("/tool/search", msg["payload"])
        await self.publish("/task/result", result)

    def on_shutdown(self):
        self.logger.info("Alice 正在优雅关闭。")
# MCP-Publish 桥接将总线 Topic 暴露为 MCP 工具
# 内置 MCP Server Node 将 publish() 抽象为 MCP Tool

from mcp import ClientSession
from tagentacle_py.mcp import TagentacleClientTransport

# 通过 Tagentacle 总线连接，而非 stdio
transport = TagentacleClientTransport(node, server_node_id="sqlite_server")
async with ClientSession(transport) as session:
    await session.initialize()
    result = await session.call_tool("query", {"sql": "SELECT * FROM users"})

🛠️ MCP 集成

Bus-as-Transport —— 以优雅架构解决 MCP Session 跨进程序列化难题。

🏠 Session 本地化

MCP Client/Server Session 留在节点内存中 —— 从不跨进程传输。Session 本地化，性能真实可靠。

🚌 总线流量转发

自定义 TagentacleTransport 将 JSON-RPC 封装为 Service 请求，在分布式环境中实现透明高效的工具调用。

🔀 双轨集成

MCP 载荷同时镜像到专用 Topic（/mcp/traffic），实现无侵入的观测和调试。

↔️ 双向调用

完整支持 MCP 规范，包括 Sampling（服务端调用 Agent）。Agent 节点注册 /rpc 服务接收回调。

👁️ 全面可观测

所有隧道流量自动镜像到 /mcp/traffic Topic。任何节点都可以审计工具调用流程，无需侵入式代理。

🔌 无缝 SDK

通过总线连接，用法与 stdio 完全一致。替换传输层即可 —— MCP 逻辑代码无需任何修改。

📜 通信协议

TCP 19999，换行分隔 JSON（JSON Lines）。简洁、快速、通用。

📡 Topic

订阅
{"op": "subscribe", "topic": "/chat/global", "node_id": "alice_node"}
发布
{"op": "publish", "topic": "/chat/global", "sender": "bob_node", "payload": {"text": "你好！"}}
消息（推送）
{"op": "message", "topic": "/chat/global", "sender": "bob_node", "payload": {"text": "你好！"}}

🔁 Service

注册
{"op": "advertise_service", "service": "/tool/read_file", "node_id": "fs_node"}
调用
{"op": "call_service", "service": "/tool/read_file", "request_id": "req-1", "payload": {"path": "a.txt"}}
响应
{"op": "service_response", "service": "/tool/read_file", "request_id": "req-1", "payload": {"content": "..."}}

🛠️ CLI 工具链

开发者的核心入口 —— 从命令行管理守护进程、包、环境和服务。

tagentacle daemon
启动本地 TCP 消息总线
tagentacle run --pkg <dir>
激活包的 .venv 并启动其 Node
tagentacle launch <config.toml>
从拓扑配置编排多节点
tagentacle topic echo <topic>
订阅并实时打印消息
tagentacle service call <srv> <json>
从命令行测试服务
tagentacle bridge --mcp <cmd>
将外部 MCP Server (stdio) 桥接到总线
tagentacle setup dep --pkg <dir>
为单个包运行 uv sync
tagentacle setup dep --all <ws>
安装所有包，生成 install/ 结构
tagentacle doctor
健康检查 —— 守护进程状态、节点连通性
# 初始化整个工作区
tagentacle setup dep --all .
# → 在每个包中运行 uv sync
# → 创建 install/src/<pkg>/.venv 符号链接
# → 生成 install/setup_env.bash

# 加载环境（将所有 .venv 添加到 PATH）
source install/setup_env.bash

# 清理
tagentacle setup clean --workspace .
